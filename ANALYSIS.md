# การวิเคราะห์และออกแบบระบบ (System Analysis and Design)

## 1. การวิเคราะห์ความต้องการของผู้ใช้ (User Requirement Analysis)

### แนวคิด (Concept)
แอปพลิเคชัน "Community Savings" (ออมสินชุมชน) พัฒนาขึ้นเพื่อส่งเสริมการออมร่วมกันภายในชุมชนหรือกลุ่มคนที่มีเป้าหมายเดียวกัน เช่น การระดมทุนเพื่อพัฒนาพื้นที่สาธารณะ การเก็บเงินเพื่อกิจกรรมกลุ่ม หรือการช่วยเหลือสมาชิกในยามฉุกเฉิน

### เหตุผลและที่มา (Reasons & Rationale)
- **ปัญหา**: การเก็บเงินกองกลางแบบเดิมมักใช้เงินสดหรือบัญชีส่วนตัว ทำให้ตรวจสอบยากและขาดความโปร่งใส
- **ทางแก้**: แอปพลิเคชันช่วยให้สมาชิกเห็นเป้าหมายร่วมกัน ติดตามยอดเงินได้แบบ Real-time และสร้างแรงจูงใจในการออมผ่านการแสดงผลความคืบหน้า (Progress Visualization)

### กลุ่มเป้าหมาย (Target Audience)
- กลุ่มชุมชนหมู่บ้าน
- กลุ่มนักเรียน/นักศึกษาที่ทำกิจกรรมร่วมกัน
- ครอบครัวที่ต้องการออมเงินเพื่อเป้าหมายร่วมกัน

## 2. การออกแบบฟีเจอร์หลัก (Feature Design)

### แผนผังระบบ (System Architecture)
ระบบทำงานแบบ Standalone บนมือถือ โดยใช้ Local Storage (Shared Preferences) ในการบันทึกข้อมูล เพื่อความง่ายและสามารถใช้งานได้แม้ไม่มีอินเทอร์เน็ต (Offline First)

### ฟีเจอร์หลัก (Key Features)
1.  **Goal Management**: สร้างและจัดการเป้าหมายการออม (ชื่อ, รายละเอียด, ยอดเงินเป้าหมาย)
2.  **Contribution Tracking**: บันทึกการจ่ายเงินสมทบของสมาชิก (ชื่อผู้จ่าย, จำนวนเงิน, วันที่)
3.  **Progress Visualization**: แสดงแถบความคืบหน้า (Progress Bar) และเปอร์เซ็นต์ความสำเร็จ
4.  **Data Persistence**: บันทึกข้อมูลลงในเครื่อง ไม่สูญหายเมื่อปิดแอป

## 3. การบริหารจัดการงาน (Team Management)

### สมาชิกในทีม (Team Members)

### การแบ่งหน้าที่ (Roles & Responsibilities)
- **Project Manager**: วางแผนภาพรวมและติดตามความคืบหน้า
- **Developer**: พัฒนาโค้ดส่วน Logic, UI และ Database
- **Designer**: ออกแบบหน้าจอ (UI/UX) และเลือกใช้โทนสี
- **Tester**: ทดสอบการใช้งานและหาข้อผิดพลาด
